# Roadmap: 트래커 마케팅 분석 강화

> 작성일: 2026-05-27
> 성격: **에픽/로드맵** (구현 X — 하위 PDCA들의 청사진)
> 상태: 진행 중 (1단계 시작 전)

## 1. 배경

이전 PDCA `tracker-overview`로 사이트 분석 개요 탭을 만들었지만, 영업/마케팅 운영 관점에서 **"어디서 새는가 / 어디까지 가는가 / 어떤 캠페인이 효과 있는가"** 같은 깊은 분석은 아직 없다.

애디온(레퍼런스) 트래커와 비교하면 다음이 부족:
- **이탈 페이지 분석** — 어느 페이지에서 사용자가 나가는지
- **세그먼트 필터** — 디바이스/유입소스로 전체 데이터 슬라이싱
- **사용자 지정 기간** — 현재는 7/30/90일 preset만
- **광고 소재 성과** — utm_content별 비교
- **전환 목표(goals)** — "특정 페이지 도달 = 목표" 정의 후 캠페인별 측정
- **퍼널** — 가입 깔때기(방문→요금제→신청→가입) 단계별 이탈률
- **기간 비교(compare)** — 두 기간 나란히 보기

## 2. 전체 목표

디하/백오피스랩/픽셀앤로직 등 모든 워크스페이스에서 **마케팅 깔때기의 누수 지점 + 캠페인 효율**을 데이터로 추적·개선할 수 있게 한다. 트래커 제품의 범용성(프로덕트별 매칭/필드 설정 불변)은 유지.

## 3. 2단계 구조

### **Phase 1: 마케팅 분석 v1** (작고 효과 큼)
- **PDCA 이름**: `tracker-marketing-v1`
- **포함 기능**:
  1. **이탈 페이지 TOP10** — 인기 페이지 옆 위젯. 사용자가 마지막으로 본 페이지 집계 (excludePaths 반영)
  2. **세그먼트 필터** — 상단에 디바이스(desktop/mobile/tablet) + 유입 채널(직접/네이버/구글/메일/광고) 드롭다운. 선택 시 KPI·차트·위젯 전부 재계산
  3. **사용자 지정 기간(DateRangePicker)** — preset 옆에 from/to 직접 선택. URL 쿼리(`from`/`to`)로 공유 가능
  4. **광고 소재(utm_content) TOP** — 캠페인 소재별 세션/전환 비교 (작은 위젯)
- **데이터**: 기존 트래커 스키마로 충분, 신규 테이블 X
- **예상 작업량**: 중간 (위젯 4개 + 필터 상태 관리 + URL sync)

### **Phase 2: 사용자정의 퍼널** (`tracker-funnels-v1`)
- 가장 큰 작업, 단독 PDCA로 분리
- **포함**: `tracker_funnels` 테이블 신설 + 단계 정의 UI + 단계별 도달률/이탈률 시각화
- 디하 도메인 박힘(가입/구독중) 본격 해결의 본체. 어떤 사이트도 자기 퍼널 정의 가능
- **데이터**: 신규 마이그레이션 1건, 정의 UI 폼

### **Phase 2.5: 기간 비교** (`tracker-compare-v1`)
- 가벼움 — KPI ±%는 이미 있고, 차트 오버레이만 추가
- **포함**: 일별 PV/가입/구독 차트에 직전 기간 비교 라인 오버레이
- DB 변경 없음, API/UI 약간 확장

### **Phase 2.6: 전환 목표 + 캠페인 트렌드** (`tracker-campaigns-v1`)
- Phase 2의 퍼널 인프라 재사용 (목표 = 단계 1개짜리 퍼널)
- **포함**: 전환 목표(goals) 위젯 + utm_campaign 시계열 차트
- 캠페인 데이터 더 쌓인 후 의미 살아남

### **(보류) Phase 3: 운영 편의**
- 알림(alerts), 클릭 룰, 디버그 스트림, 이벤트 탭
- 당장 필요도 낮음. Phase 1·2 운영 본 후 재평가.

## 4. 의존성 / 순서

```
[Phase 0] tracker-overview ✅
   ↓
[Phase 1] tracker-marketing-v1 ✅
   ↓ (필터·기간 인프라 재사용)
[Phase 2] tracker-funnels-v1  ← 다음 (사용자정의 퍼널)
   ↓ (퍼널 정의 모델 재사용)
[Phase 2.5] tracker-compare-v1 (기간 비교)
   ↓
[Phase 2.6] tracker-campaigns-v1 (목표 + 캠페인 트렌드)
   ↓
[Phase 3] 운영 편의 (알림/클릭룰/디버그/이벤트 탭 — 보류)
```

**중요 원칙**:
- Phase 1·2 다 **단일 사이트 기준**으로 시작. multi-site 합산은 별도 과제 (모든 ws당 site 1개라 현실적 영향 적음).
- 모든 분석은 **excludePaths 반영** (마케팅 깔때기 진짜 수치). tracker-overview에서 만든 규칙 그대로 상속.
- 제품/필드명 하드코딩 0건 — 사이트 설정 기반 (matchField, excludePaths).
- 외부 트래커 스크립트 페이로드 스펙 불변 — 고객사 무수정.

## 5. 비범위 (이 로드맵)

- 트래커 스크립트 자체 변경 (이벤트 추가 등)
- AI 기반 자동 인사이트 / 추천
- 회사 단위 묶기 (같은 도메인 다른 담당자 통합 — 별도 과제)
- 트래커 외 다른 분석(이메일·CRM 등)

## 7. 사용 안내

- 이 문서는 **에픽(영구 보관)** — `/pdca` 사이클을 타지 않는다 (plan에서 멈춤).
- 각 Phase 시작 시 별도 PDCA: `/pdca plan tracker-marketing-v1` → design → do → analyze → report → archive.
- Phase 완료 후 이 로드맵의 "진척 추적" 표만 업데이트.
- 우선순위/범위 변경 발생 시 이 문서 갱신.

## 8. 결정 사항 (Phase 1 design에서 확정)

다음 사항은 Phase 1 design 단계에서 정한다 — 미리 정하지 않음:
- 세그먼트 필터 동작: AND vs OR (다중 선택 시)
- DateRangePicker 컴포넌트 (이미 있는지/새로 만들지)
- 광고 소재 위젯 위치 (사이드 vs 하단)
- URL 쿼리 sync 범위 (기간만 vs 필터까지)
