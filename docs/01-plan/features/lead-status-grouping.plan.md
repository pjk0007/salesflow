# Plan: 리드 상태별 그룹 뷰

> 레코드(리드) 목록을 상태별로 그룹핑하여 노션 데이터베이스처럼 보여주는 기능

## 1. 배경 & 문제

- 현재 레코드 목록은 상태에 관계없이 **플랫 테이블**로 나열
- 리드가 늘어날수록 "신규", "연락 중", "핵심", "종료" 등 상태별 현황 파악이 어려움
- 사용자가 원하는 것: **노션 데이터베이스의 그룹 뷰**처럼 상태별로 접고/펼 수 있는 그룹 테이블

## 2. 목표

| 항목 | 설명 |
|------|------|
| 핵심 가치 | 리드를 상태별로 한눈에 파악, 관리 효율 향상 |
| 대상 사용자 | Salesflow를 사용하는 영업 담당자 / 관리자 |
| 성공 지표 | 그룹 뷰 활성화 시 상태별 리드 수 즉시 확인 가능 |

## 3. 기능 범위

### 3.1 In-Scope (이번에 구현)

1. **상태 필드 기준 그룹핑**
   - `selectWithStatusBg` 타입 필드(상태 필드)를 기준으로 레코드를 그룹화
   - 각 그룹은 상태 라벨 + 배지 색상 + 레코드 수 표시
   - 그룹 접기/펼치기 (토글)

2. **그룹 뷰 토글**
   - 기존 플랫 테이블 뷰 ↔ 그룹 뷰 전환 버튼 (툴바에 추가)
   - 사용자 선택 기억 (localStorage 또는 파티션 설정)

3. **그룹 내 테이블**
   - 각 그룹 안에서는 기존 테이블과 동일한 컬럼/인라인 편집/정렬 유지
   - 그룹 헤더: `▼ ● 연락 중  41` 형태 (이미지 참고)

4. **"+ 신규 Item" 버튼**
   - 각 그룹 하단에 해당 상태로 바로 레코드 추가 버튼

### 3.2 Out-of-Scope (이번에는 안 함)

- 칸반 보드 뷰 (드래그 앤 드롭 카드 형태)
- 임의 필드 기준 그룹핑 (상태 필드 외)
- 그룹 간 드래그 앤 드롭 이동
- 서버사이드 그룹핑/페이지네이션 (클라이언트에서 처리)

## 4. 현재 구조 분석

### DB 스키마
- `records.data` (JSONB) 안에 상태값 저장
- `statusOptionCategories` → `statusOptions` 테이블로 상태 옵션 관리
- `fieldDefinitions.statusOptionCategoryId`로 필드-상태카테고리 연결
- `statusOptions`에 `bgColor`, `sortOrder` 존재

### 컴포넌트 구조
| 파일 | 역할 |
|------|------|
| `app/records/page.tsx` | 페이지 조립 |
| `components/records/RecordTable.tsx` | 플랫 테이블 (현재) |
| `components/records/RecordToolbar.tsx` | 툴바 (검색/필터/내보내기) |
| `hooks/useRecords.ts` | 레코드 데이터 페칭 |
| `components/records/CellRenderer.tsx` | 셀 렌더링 |

### API
- `GET /api/partitions/[id]/records` → 플랫 리스트 반환 (페이지네이션)

## 5. 구현 방향

### 접근 방식: 클라이언트사이드 그룹핑

- API는 기존대로 플랫 리스트 반환
- 클라이언트에서 상태 필드 기준으로 레코드를 그룹핑
- 그룹 뷰일 때는 **전체 레코드를 한번에 조회** (페이지네이션 대신)하거나, 기존 페이지네이션 유지하되 가져온 데이터 내에서 그룹핑

### 새로 만들 컴포넌트

```
src/components/records/
├── GroupedRecordView.tsx    # 그룹 뷰 컨테이너
├── RecordGroup.tsx          # 개별 그룹 (헤더 + 테이블)
└── (기존 파일들 수정)
    ├── RecordToolbar.tsx    # 그룹 뷰 토글 버튼 추가
    └── RecordTable.tsx      # 그룹 내 테이블로도 재사용
```

### 변경할 파일
- `app/records/page.tsx` — 뷰 모드 상태 관리, 조건부 렌더링
- `hooks/useRecords.ts` — 그룹 뷰 시 pageSize 조정 (또는 전체 조회)

## 6. 참고 이미지 분석

노션 스타일 그룹 뷰:
- 그룹 헤더: `▼ ● {상태명}  {개수}` (접기/펼치기 가능)
- 그룹 내 테이블: 기존 컬럼 헤더 유지 (상태, 출처, 소재, 체험 요청 등)
- 그룹 하단: `+ 신규 Item` 추가 버튼
- 그룹 사이에 약간의 간격

## 7. 리스크 & 고려사항

| 리스크 | 대응 |
|--------|------|
| 레코드 수가 많으면 전체 조회 시 성능 문제 | 초기에는 기존 페이지네이션 내 그룹핑, 추후 서버사이드 그룹핑 고려 |
| 상태 필드가 없는 파티션 | 그룹 뷰 버튼 비활성화 또는 숨김 |
| 값이 없는 레코드 | "미분류" 그룹으로 모아서 표시 |
