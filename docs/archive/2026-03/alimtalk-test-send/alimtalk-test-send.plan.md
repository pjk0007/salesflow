# Plan: alimtalk-test-send (알림톡 템플릿 테스트 발송)

## 1. 개요

### 배경
현재 알림톡 발송은 레코드 기반(templateLink → 레코드 필드에서 수신번호/변수 자동 매핑)으로만 가능.
승인된 템플릿을 실제 발송해보려면 더미 레코드를 만들어야 하는 불편함이 있음.

### 목표
승인된 알림톡 템플릿에 대해 수신번호와 변수값을 직접 입력하여 테스트 발송할 수 있는 기능 추가.

## 2. 현재 상태

### 기존 발송 흐름
1. 템플릿 링크(alimtalk_template_links) 생성 — 파티션, 수신필드, 변수매핑 설정
2. 레코드 선택 → SendAlimtalkDialog에서 발송
3. 레코드 필드에서 수신번호/변수값 자동 추출 → NHN API 발송

### 부족한 점
- 레코드 없이 직접 수신번호/변수 입력 불가
- 승인 후 실제 발송 확인에 더미 레코드 필요
- NHN 콘솔에서는 테스트 발송 기능 제공하나 현재 UI에서 미지원

## 3. 구현 범위

### 기능
1. **테스트 발송 다이얼로그** — 템플릿 목록에서 "테스트 발송" 버튼
2. **수신번호 직접 입력** — 1개 번호 입력
3. **변수값 직접 입력** — 템플릿 본문에서 `#{변수명}` 추출 → 각 변수에 대한 입력 필드 자동 생성
4. **미리보기** — 변수 치환된 최종 본문 표시
5. **발송 결과** — 성공/실패 표시

### 진입점
- 템플릿 목록 테이블의 각 행에 "테스트 발송" 버튼 (승인 상태일 때만 활성)
- 또는 템플릿 상세/수정 페이지에서 "테스트 발송" 버튼

## 4. 변경 파일

| 파일 | 변경 |
|------|------|
| `src/components/alimtalk/TestSendDialog.tsx` | 신규 — 테스트 발송 다이얼로그 |
| `src/app/api/alimtalk/test-send/route.ts` | 신규 — 테스트 발송 API |
| `src/components/alimtalk/TemplateList.tsx` | 테스트 발송 버튼 추가 |

### API route 불필요?
기존 NhnAlimtalkClient.sendMessages()는 senderKey + templateCode + recipientList(수신번호+변수)를 받으므로,
테스트 발송도 동일한 API를 사용 가능. 다만 templateLink 없이 직접 senderKey/templateCode를 받는 전용 API가 필요.

## 5. 검증
- `npx next build` 성공
- 승인된 템플릿에서 테스트 발송 버튼 표시
- 수신번호 + 변수값 입력 후 발송 성공
- 변수가 없는 템플릿도 발송 가능
- 발송 결과(성공/실패) 표시
