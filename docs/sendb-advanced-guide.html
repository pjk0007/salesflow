1. 레코드 고급 관리

중복 관리

같은 고객이 중복 등록되는 것을 방지하고 관리합니다.

파티션 중복 설정 (파티션 우클릭 → 배분 설정)

설정설명
duplicateCheckField중복 판별 기준 필드 (예: email, phone)
중복 처리 정책 reject — 중복 레코드 등록 거부
allow — 중복 허용 (색상 하이라이트로 표시)
merge — 기존 레코드에 병합
delete_old — 기존 레코드 삭제 후 새로 등록
하이라이트 색상중복 레코드 행 배경색 지정 (기본: 연한 노랑)
CSV 가져오기 시에도 중복 체크가 적용됩니다. 기준 필드를 미리 설정해두면 대량 등록 시 중복을 자동으로 처리합니다.

그룹 뷰 활용

select 타입 필드에 isGroupable을 켜면 레코드를 상태별 칸반 스타일로 볼 수 있습니다.

설정 방법

  1. /settings/workspace → 워크스페이스 선택 → 필드 관리
  2. select 타입 필드 (예: "상태") 편집 → "그룹화 가능" 체크
  3. 옵션 추가 (예: 신규, 연락중, 미팅, 계약, 이탈)
  4. /records 상단 뷰 모드를 "그룹"으로 전환

각 그룹 헤더에 건수가 표시되며, 그룹 내에서도 인라인 편집, 선택, 정렬이 모두 동작합니다.

필터 조합 활용

필터는 여러 개를 동시에 추가할 수 있으며 AND 조건으로 동작합니다.

연산자사용 예시
같음 (=)상태 = "신규"
같지 않음 (≠)상태 ≠ "이탈"
포함회사 포함 "테크"
비어있음이메일 비어있음 → 이메일 없는 레코드만
비어있지 않음전화번호 비어있지 않음 → 전화번호 있는 레코드만
이상/이하매출액 이상 1000000
활용 예: "상태 = 신규" + "이메일 비어있지 않음" 필터 → 이메일 발송 가능한 신규 리드만 추출 → 체크박스 전체 선택 → 이메일 발송

컬럼 표시 관리

파티션마다 보이는 컬럼을 다르게 설정할 수 있습니다. 툴바의 컬럼 아이콘을 클릭하여 각 필드의 체크박스를 토글합니다. 이 설정은 파티션별로 저장됩니다.

2. 파티션 운영 전략

폴더로 파이프라인 구성

예시: B2B 영업 파이프라인
📁 인바운드
  ├── 웹폼 유입
  └── 광고 유입
📁 영업 진행
  ├── 1차 미팅
  ├── 제안서 발송
  └── 계약 협상
📁 완료
  ├── 계약 성사
  └── 이탈/보류

파티션은 드래그 앤 드롭으로 폴더 간 이동이 가능합니다. 영업 단계가 변할 때 레코드를 파티션 간에 이동하는 대신, 그룹 뷰의 상태 필드를 변경하는 것이 더 편리합니다.

배분(라운드로빈) 설정

신규 리드를 팀원에게 균등 배분하는 기능입니다.

  1. 파티션 우클릭 → "배분 설정"
  2. 배분 활성화 ON
  3. 배분 인원 수 설정 (예: 3명이면 순서 1, 2, 3이 순환)
  4. 배분 기준 필드 지정 (어떤 필드에 담당자 순번을 기록할지)

이후 새 레코드가 등록되면 자동으로 순서가 부여됩니다. 레코드 툴바에서 배분 순서별 필터링이 가능합니다.

팀 활용: 각 팀원이 자기 배분 순서만 필터링하면 자신의 담당 리드만 볼 수 있습니다.

이메일 중복 발송 방지

이메일 연결(자동발송)에서 preventDuplicate 옵션을 ON으로 설정하면 같은 레코드에 같은 템플릿을 중복 발송하지 않습니다.

이미 발송된 이력이 있으면 트리거가 발생해도 건너뛰고 로그에 기록합니다.

3. 이메일 자동화 심화

자동발송 트리거 비교

트리거발동 시점적합한 용도
레코드 생성새 레코드가 추가될 때환영 이메일, 리드 확인 안내
레코드 수정기존 레코드 데이터 변경 시상태 변경 알림, 단계별 안내
반복 발송설정한 주기마다뉴스레터, 주간 리포트

후속 이메일 자동 발송

수신자가 이메일을 읽은 후 일정 시간이 지나면 자동으로 후속 이메일을 보냅니다.

이메일 발송 → 수신자 읽음 → N일 대기 → 후속 이메일 자동 발송

설정 방법

  1. 이메일 연결 편집 페이지 하단 "후속 이메일 설정" 섹션 열기
  2. 활성화 ON
  3. 대기 일수 설정 (예: 3일)
  4. 후속 이메일 모드 선택:
    • 템플릿 모드 — 미리 만든 템플릿으로 발송
    • AI 모드 — 원본 이메일 컨텍스트를 참고하여 AI가 후속 메시지 생성
후속 큐 현황은 /email → "후속 큐" 탭에서 확인할 수 있습니다. 상태(대기/발송완료/취소)와 예정 시각이 표시됩니다.

클릭 추적 활용

연결 설정에서 "클릭 추적"을 ON으로 하면 이메일 본문의 모든 링크에 추적 코드가 자동 삽입됩니다.

추적 대상: <a href="..."> 태그의 일반 URL (mailto:, tel:, #앵커, javascript: 제외)

동작 원리:

수신자 클릭 → 추적 서버 기록 → 원본 URL로 리다이렉트

확인 위치:

  • 발송 이력 테이블 → 클릭수 배지 (파란색)
  • 대시보드 → 클릭률 카드 + 발송유형별 클릭률
  • 홈 → 발송 규칙별 성과 Top 10 → 클릭률 컬럼

AI 자동발송 vs 템플릿 자동발송

템플릿 자동발송
  • 미리 만든 템플릿 사용
  • #{변수}로 개인화
  • 모든 수신자에게 같은 구조
  • AI 불필요 (API 키 없어도 OK)
  • 발송 속도 빠름
AI 자동발송
  • AI가 수신자마다 맞춤 이메일 생성
  • 기업 조사 결과 + 제품 정보 참고
  • 수신자마다 다른 내용
  • AI API 키 필요
  • 생성 시간 소요 (몇 초)
추천 전략: 대량 안내성 메시지 → 템플릿 자동발송, 1:1 맞춤 영업 메일 → AI 자동발송

기능위치용도

AI 프롬프트 작성 팁

나쁜 예

"이메일 써줘"

"좋은 이메일 만들어"

좋은 예

"B2B SaaS 제품을 처음 소개하는 콜드메일. 간결하게 3문단, 상대방의 페인포인트를 먼저 언급하고 우리 솔루션으로 연결. CTA는 15분 미팅 제안."

효과적인 프롬프트 구성 요소:

  • 목적 — 어떤 종류의 메시지인지 (소개, 팔로업, 안내, 감사)
  • 톤 — 격식체/비격식체, 전문적/친근한
  • 구조 — 문단 수, 길이 제한
  • CTA — 수신자에게 원하는 다음 행동
  • 컨텍스트 — 업종, 상황, 관계

AI + 제품 정보 시너지

제품 등록 → AI 자동발송 규칙에 "제품 참고" ON → AI가 제품 특장점을 반영한 맞춤 이메일 생성

제품 관리에 등록한 정보(이름, 소개, 가격, 특장점)가 AI 프롬프트 컨텍스트에 포함되어 더 정확한 영업 메일이 생성됩니다.

AI + 기업 조사 시너지

레코드 생성 (회사명 포함) → AI 기업 조사 자동 실행 → 수집된 기업 정보를 AI 이메일에 반영

AI 자동발송 규칙에서 "기업 조사 자동 실행"을 켜면 레코드의 회사명으로 웹 검색을 수행하고, 그 결과를 이메일 생성 컨텍스트에 포함합니다.

변수 활용

본문에 #{변수명}을 넣으면 발송 시 레코드 데이터로 치환됩니다.

연결 관리에서 변수 매핑을 설정합니다: 템플릿 변수 ↔ 레코드 필드를 1:1 연결.

검수 주의사항: 카카오 검수에서 변수는 최소 1개의 예시값을 포함해야 합니다. 검수 코멘트에 "#{이름} → 홍길동, #{금액} → 50,000원" 형태로 작성하세요.

검수 프로세스

템플릿 등록 (TSC01) → 검수 요청 (코멘트 작성) → 검수중 (TSC02) → 승인 (TSC03)

반려(TSC04) 시 사유를 확인하고 수정 후 재검수 요청합니다. 승인된 템플릿만 실제 발송이 가능합니다.

7. 팀 운영과 권한

역할(Role) 체계

역할레코드발송설정 변경팀 관리요금제
Owner전체전체전체전체전체
Admin전체전체전체초대/제거변경 가능
Member전체전체불가불가불가

역할은 조직별로 부여됩니다. 같은 사용자가 A조직에서는 Admin, B조직에서는 Member일 수 있습니다.

팀원 초대 방법

  1. /settings/organization → "팀멤버" 탭
  2. 이메일 입력 + 역할 선택 → 초대
  3. 초대받은 사용자는 이메일로 초대 링크를 받습니다
  4. 이미 가입된 사용자는 로그인 시 조직이 자동 추가됩니다
  5. 미가입 사용자는 가입 후 조직에 참여합니다

팀 운영 팁

  • 배분 기능으로 리드를 팀원에게 자동 분배
  • 각 팀원은 배분 순서 필터로 자기 담당만 확인
  • Admin 2명 이상 권장 (한 명 부재 시 설정 변경 가능)
  • Member 역할로 실수로 인한 설정 변경 방지

8. 외부 연동 (API)

API 토큰 관리

/settings/organization → "API 토큰" 탭에서 Bearer 토큰을 발급합니다.

스코프(범위) 설정

스코프접근 범위
파티션특정 파티션의 레코드만
폴더폴더 내 모든 파티션의 레코드
워크스페이스워크스페이스 내 모든 레코드
보안 주의: 토큰은 생성 직후에만 전체 값이 표시됩니다. 안전한 곳에 보관하세요. 유출된 토큰은 즉시 삭제하고 새로 발급하세요.

외부 API 사용법

기본 엔드포인트

GET /api/v1/records — 레코드 목록 조회

POST /api/v1/records — 레코드 생성

PUT /api/v1/records/:id — 레코드 수정

DELETE /api/v1/records/:id — 레코드 삭제

인증

Authorization: Bearer sk_xxxxxxxxxxxxxxxxxxxx

활용 예시

  • 외부 서비스(Zapier, Make 등)에서 레코드 자동 생성
  • 자사 시스템과 고객 데이터 양방향 동기화
  • 커스텀 대시보드에서 레코드 데이터 조회

9. 멀티 조직 운영

조직 구조

조직 (Organization) → 워크스페이스 → 폴더 → 파티션 → 레코드

한 사용자가 여러 조직에 소속될 수 있습니다. 사이드바 상단의 조직 전환 드롭다운으로 전환합니다.

멀티 조직이 유용한 경우

  • 에이전시 — 클라이언트별로 조직을 분리하여 데이터 격리
  • 부서별 관리 — 영업부/마케팅부를 조직 단위로 분리
  • 프리랜서 — 개인 프로젝트와 고객 프로젝트를 분리

조직 전환

사이드바 상단 드롭다운에서 다른 조직을 선택하면 JWT 토큰이 재발급되며, 해당 조직의 데이터로 전환됩니다. 모든 페이지의 데이터가 선택한 조직 기준으로 표시됩니다.

10. 실전 시나리오

시나리오 A: 인바운드 리드 자동 처리
웹 폼 제출 → 레코드 자동 생성 → 환영 이메일 자동 발송 → 3일 후 읽음 시 후속 이메일 → 배분으로 담당자 배정

설정 순서

  1. "신규 리드" 파티션 생성 + 배분 설정 (팀원 3명)
  2. 이메일 템플릿 생성 (환영 + 후속)
  3. 이메일 연결: 파티션="신규 리드" + 트리거="생성 시" + 후속 이메일 3일
  4. 웹 폼 생성: 대상="신규 리드" 파티션
  5. 홈페이지에 웹 폼 임베드

결과: 방문자가 폼을 제출하면 → 레코드 생성 → 환영 이메일 발송 → 읽으면 3일 후 후속 → 담당 팀원에게 자동 배분. 수동 작업 제로.

시나리오 B: AI 맞춤 콜드메일 캠페인
잠재고객 CSV 업로드 → AI 기업 조사 자동 실행 → AI가 수신자별 맞춤 이메일 생성+발송 → 클릭 추적으로 관심도 측정

설정 순서

  1. AI API 키 등록 (조직 설정 → AI)
  2. 제품 정보 등록 (제품 관리)
  3. "콜드메일 대상" 파티션 생성
  4. AI 자동발송 규칙: 파티션="콜드메일 대상" + 트리거="생성 시" + 기업 조사 ON + 제품 참고 ON
  5. CSV로 잠재고객 일괄 등록

결과: CSV 업로드만으로 각 잠재고객에게 회사 정보를 반영한 개인화된 콜드메일이 자동 발송됩니다.

시나리오 C: 알림톡 + 이메일 멀티채널 캠페인
레코드 상태 "계약" 변경 → 알림톡: 계약 완료 알림
레코드 상태 "계약" 변경 → 이메일: 상세 계약서 + 가이드

설정 순서

  1. 알림톡 템플릿 생성 → 검수 승인
  2. 이메일 템플릿 생성 (계약 완료 안내)
  3. 알림톡 연결: 트리거="수정 시" + 조건="상태 = 계약"
  4. 이메일 연결: 트리거="수정 시" + 조건="상태 = 계약"

결과: 상태만 변경하면 알림톡(즉시 알림)과 이메일(상세 안내)이 동시에 발송됩니다.
